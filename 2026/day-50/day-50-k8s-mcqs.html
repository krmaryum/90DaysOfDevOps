Day 50 — Kubernetes MCQs

25 practice questions based on your Day 50 Kubernetes notes: architecture, control plane, worker nodes, kubeconfig, kind, kubectl, lifecycle, and kube-system components.

Q1. Why was Kubernetes created?

Explanation: Docker can run containers, but Kubernetes was created to orchestrate many containers across multiple machines.

Q2. Which company originally created Kubernetes?

Explanation: Kubernetes was created by Google and inspired by its internal systems.

Q3. Kubernetes was inspired by which internal Google system?

Explanation: Kubernetes was inspired by Google's internal cluster manager called Borg.

Q4. What does the word 'Kubernetes' mean?

Explanation: Kubernetes means 'helmsman' or ship captain.

Q5. What is the main entry point to a Kubernetes cluster?

Explanation: All cluster commands go through the API Server.

Q6. What is etcd used for in Kubernetes?

Explanation: etcd is the key-value store that holds the cluster's state.

Q7. Which control plane component decides where a new Pod should run?

Explanation: The Scheduler selects the most suitable node for a new Pod.

Q8. What does the Controller Manager do?

Explanation: The Controller Manager continuously works to keep the cluster in the desired state.

Q9. What is the smallest deployable unit in Kubernetes?

Explanation: A Pod is the smallest deployable unit in Kubernetes.

Q10. Kubernetes mainly manages which object directly?

Explanation: Kubernetes works with Pods, not individual containers directly.

Q11. What is meant by 'desired state' in Kubernetes?

Explanation: You define what you want, and Kubernetes works to keep the cluster in that state.

Q12. When you run 'kubectl apply -f pod.yaml', which component receives the request first?

Explanation: kubectl sends the request to the Kubernetes API Server.

Q13. Which node component talks to the API Server and manages Pods on that node?

Explanation: kubelet is the node agent that watches the API Server and manages local Pods.

Q14. What does kube-proxy mainly handle?

Explanation: kube-proxy manages networking rules so Pods and Services can communicate.

Q15. What is the job of the container runtime on a worker node?

Explanation: The container runtime, such as containerd or CRI-O, starts and runs containers.

Q16. What usually happens if the API Server goes down?

Explanation: Without the API Server, new operations fail, but existing workloads may keep running for some time.

Q17. What happens if a worker node goes down?

Explanation: Kubernetes detects the mismatch and recreates Pods on healthy nodes if needed.

Q18. If etcd data is completely lost and no backup exists, what is the best description?

Explanation: etcd stores the cluster state. Losing it means losing the cluster's memory.

Q19. What is kubectl?

Explanation: kubectl is the command-line tool used to interact with a Kubernetes cluster.

Q20. Where is kubeconfig typically stored on Linux?

Explanation: The default kubeconfig location is ~/.kube/config.

Q21. What does a kubeconfig file contain?

Explanation: kubeconfig stores how kubectl connects and authenticates to clusters.

Q22. How does kubectl know which cluster to use when multiple clusters exist?

Explanation: kubectl uses the current context defined in kubeconfig.

Q23. Which local Kubernetes option did you choose for Day 50 and why?

Explanation: kind is lightweight and runs Kubernetes nodes as Docker containers, which makes it great for local learning.

Q24. What does the command 'kubectl get pods -A' do?

Explanation: The -A flag means all namespaces.

Q25. Which of the following commonly appears in the 'kube-system' namespace?

Explanation: The kube-system namespace contains important cluster system components.
