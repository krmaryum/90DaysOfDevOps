# Task 5 – Clean Up Your GitHub Profile

## Task Overview

Over time, GitHub accounts accumulate old repositories, incomplete experiments, abandoned projects, and files that no longer represent current skills or career goals.

The purpose of this task is to review your repositories and remove anything that creates a poor impression while ensuring no sensitive information is publicly exposed.

A clean GitHub profile appears more professional and makes it easier for recruiters to focus on your best work.

---

# Task Objectives

By completing this task, you will:

* Remove unnecessary repositories.
* Archive inactive projects.
* Rename unclear repositories.
* Protect sensitive information.
* Improve repository organization.
* Create a professional GitHub presence.

---

# Step 1 – Review All Repositories

Visit:

```text
https://github.com/krmaryum?tab=repositories
```

Review every repository and categorize it.

---

# Repository Categories

## Keep

Repositories that:

* Demonstrate technical skills.
* Support your DevOps journey.
* Contain completed projects.
* Include documentation and READMEs.

Examples:

* 90-days-of-devops
* github-actions-kubernetes-masterclass
* shell-scripts
* python-scripts
* devops-notes
* Kubernetes projects
* Terraform projects

---

## Archive

Repositories that:

* Are old but may be useful later.
* Are no longer actively maintained.
* Have historical value.

Examples:

* Practice repositories
* Older learning projects
* Experimental repositories

### Benefits

* Keeps project history.
* Removes clutter.
* Preserves work.

---

## Delete

Repositories that:

* Are completely empty.
* Were created by mistake.
* Have no future value.
* Duplicate another repository.

### Examples

* test-repo
* temp-repo
* practice123
* unused repositories

# Step 3 – Check for Exposed Secrets

Review repositories for sensitive information.

---

## Never Commit

* Passwords
* API Keys
* Access Tokens
* AWS Credentials
* SSH Private Keys
* Database Credentials
* .env Files

Examples:

```text
.env
.env.local
.env.production
id_rsa
id_ed25519
aws-credentials.txt
```

---

# AWS Secrets to Protect

Never expose:

```text
AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
```

---

# Docker Secrets to Protect

Never expose:

```text
DOCKERHUB_USERNAME
DOCKERHUB_TOKEN
```

---

# GitHub Secrets to Protect

Never expose:

```text
GITHUB_TOKEN
PAT Tokens
```

---
