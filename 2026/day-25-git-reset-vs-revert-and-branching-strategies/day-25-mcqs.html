Day 25 MCQs

Git Reset vs Revert & Branching Strategies

25 Multiple Choice Questions for Practice

Instructions: Select one answer for each question, then click “Submit Quiz” to check your score.

1. What is the main purpose of git reset?

Correct Answer: B — git reset moves HEAD to a previous commit.

2. Which git reset option keeps changes staged?

Correct Answer: C — git reset --soft keeps changes staged.

3. What does git reset --mixed HEAD~1 do?

Correct Answer: B — --mixed removes the commit and unstages the changes.

4. Which git reset option is destructive?

Correct Answer: C — git reset --hard can delete working directory changes.

5. What command shows the history of where HEAD has been?

Correct Answer: B — git reflog shows HEAD movement history.

6. What is the main purpose of git revert?

Correct Answer: B — git revert creates a new commit that reverses changes.

7. Does git revert remove the original commit from history?

Correct Answer: B — git revert keeps the original commit in history.

8. Why is git revert safer for shared branches?

Correct Answer: C — revert preserves history and records the undo as a new commit.

9. Which command should usually be avoided on already pushed commits?

Correct Answer: B — reset rewrites history and can affect teammates.

10. What should you run after resolving a revert conflict?

Correct Answer: A — after staging resolved files, run git revert --continue.

11. Before git revert --continue, what should be done with resolved files?

Correct Answer: B — resolved conflict files should be staged with git add.

12. Which command cancels an in-progress revert?

Correct Answer: A — git revert --abort cancels an in-progress revert.

13. In GitFlow, which branch usually contains production-ready code?

Correct Answer: C — main usually stores production-ready code.

14. In GitFlow, feature branches are usually created from which branch?

Correct Answer: A — feature branches usually start from develop in GitFlow.

15. Which GitFlow branch is used for urgent production fixes?

Correct Answer: A — hotfix branches are used for urgent production fixes.

16. GitFlow is best suited for which type of team?

Correct Answer: A — GitFlow works well for large teams with planned releases.

17. What is the main idea of GitHub Flow?

Correct Answer: B — GitHub Flow uses main plus short-lived feature branches.

18. In GitHub Flow, the main branch should usually be:

Correct Answer: B — main should stay deployable in GitHub Flow.

19. GitHub Flow is commonly used by:

Correct Answer: A — GitHub Flow is useful for fast delivery.

20. What is Trunk-Based Development?

Correct Answer: A — Trunk-Based Development focuses on frequent integration into main/trunk.

21. Trunk-Based Development usually works best with:

Correct Answer: B — it depends heavily on CI/CD and strong testing.

22. Which branching strategy is best for a startup shipping fast?

Correct Answer: A — GitHub Flow is simple and fast for startups.

23. Which branching strategy is best for a large team with scheduled releases?

Correct Answer: A — GitFlow supports structured release cycles.

24. What is a major risk of long-lived branches?

Correct Answer: A — long-lived branches can cause more merge conflicts.

25. Which command is most useful as a safety net after a reset mistake?

Correct Answer: A — git reflog can help recover lost commits.
