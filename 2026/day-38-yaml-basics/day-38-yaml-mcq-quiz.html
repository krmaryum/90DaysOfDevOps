Day 38 – YAML Basics

25 MCQ Quiz

Topics Covered: YAML syntax, key-value pairs, lists, nested objects, multi-line strings, validation, indentation, and common YAML mistakes.

Q1. What does YAML currently stand for?

Explanation: YAML currently stands for YAML Ain't Markup Language, meaning it is focused on data representation, not document markup.

Q2. What is YAML mainly used for in DevOps?

Explanation: YAML is widely used to write configuration files for DevOps tools and CI/CD pipelines.

Q3. Which character should never be used for YAML indentation?

Explanation: YAML uses spaces only for indentation. Tabs can cause validation errors.

Q4. What is the standard indentation commonly used in YAML?

Explanation: Two spaces are commonly used as the standard indentation in YAML.

Q5. Which YAML example shows a correct key-value pair?

Explanation: YAML key-value pairs use the format key: value.

Q6. In YAML, what type is this value? learning: true

Explanation: true without quotes is treated as a Boolean value.

Q7. In YAML, what type is this value? learning: "true"

Explanation: When true is placed inside quotes, YAML treats it as a string.

Q8. Which value is most likely interpreted as a number in YAML?

Explanation: A number without quotes, such as 4, is interpreted as a numeric value.

Q9. Which syntax represents a block style list in YAML?

Explanation: Block style lists use dashes under a key.

Q10. Which syntax represents an inline style list in YAML?

Explanation: Inline lists are written on one line inside square brackets.

Q11. What does indentation create in nested YAML objects?

Explanation: Indentation creates hierarchy and parent-child relationships in YAML.

Q12. In this structure, what is credentials?
database:
credentials:
user: admin

Explanation: credentials is nested under database, so it is a nested object.

Q13. Which command displays the content of person.yaml?

Explanation: The cat command is used to display file content in Linux.

Q14. Which tool can be used to validate YAML online?

Explanation: yamllint.com can be used to validate YAML syntax online.

Q15. Which command installs yamllint on Ubuntu/WSL?

Explanation: On Ubuntu/WSL, yamllint can be installed using sudo apt install yamllint -y.

Q16. Which command installs yamllint on RHEL/Rocky Linux?

Explanation: On RHEL/Rocky Linux, yamllint can be installed using sudo dnf install yamllint -y.

Q17. What does the YAML literal block style | do?

Explanation: The | style preserves line breaks exactly as written.

Q18. What does the YAML folded block style > do?

Explanation: The > style folds multiple lines into a single paragraph.

Q19. When should you use | in YAML?

Explanation: Use | when line breaks must be preserved, such as scripts and commands.

Q20. When should you use > in YAML?

Explanation: Use > for long text, descriptions, notes, or documentation paragraphs.

Q21. What is wrong with this YAML?
tools:
- docker
- kubernetes

Explanation: Both list items must be aligned consistently.

Q22. Which corrected version is best practice?

Explanation: Using two spaces before each dash is clear and commonly used.

Q23. Which YAML line should be quoted because it contains a colon?

Explanation: A colon inside a string can confuse YAML, so quotes are recommended.

Q24. Which file practiced key-value pairs and lists in Day 38?

Explanation: person.yaml was used for key-value pairs and lists.

Q25. Which file practiced nested objects and multi-line strings in Day 38?

Explanation: server.yaml was used for nested objects and multi-line strings.
